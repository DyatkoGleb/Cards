# Feature Specification: Flashcard Learning App

**Feature Branch**: `001-flashcard-learning-app`  
**Created**: 2026-03-30  
**Status**: Draft  
**Input**: User description: "необходимо разработать мобильное приложение для изучения английских
слов и выражений через карточки, систему веса слова, профиль со статистикой, папки слов, выбор
набора для обучения, проверку предложений через Gemini, локальное хранение, светлую и темную темы"

## Clarifications

### Session 2026-03-30

- Q: Как вес слова влияет на частоту показа карточек? → A: Вес напрямую определяет частоту показа: чем меньше вес, тем чаще карточка.
- Q: Какое правило уникальности слов использовать при папках? → A: Уникальность по паре слово+перевод глобально; одна карточка может принадлежать нескольким папкам.
- Q: Что делать при повреждении локальных данных? → A: Пытаться восстановить частично; если невозможно — сброс только поврежденного раздела с уведомлением.

### User Story 1 - Изучение карточек (Priority: P1)

Как пользователь, я хочу повторять слова с помощью карточек, чтобы быстрее запоминать английские
слова и выражения. Я вижу либо слово, либо перевод, могу открыть вторую сторону по нажатию и
оценить, угадал(а) ли я, с помощью кнопок или свайпа.

**Why this priority**: Это основная ценность продукта; без нее приложение не выполняет свою
обучающую задачу.

**Independent Test**: Добавить минимум 10 слов, запустить сессию карточек и подтвердить:
переключение стороны карточки, случайный старт со слова/перевода (50/50), изменение веса через
кнопки и свайпы, соблюдение границ веса 0..100.

**Acceptance Scenarios**:

1. **Given** пользователь открыл экран карточек и в наборе есть слова, **When** показывается новая
   карточка, **Then** в 50% случаев первой стороной отображается слово, а в 50% случаев перевод.
2. **Given** пользователь видит первую сторону карточки, **When** он нажимает на карточку,
   **Then** отображается вторая сторона этой же карточки.
3. **Given** текущий вес слова равен 10, **When** пользователь отмечает "угадал" кнопкой или
   свайпом вправо, **Then** вес становится 11.
4. **Given** текущий вес слова равен 10, **When** пользователь отмечает "не угадал" кнопкой или
   свайпом влево, **Then** вес становится 7.
5. **Given** текущий вес слова равен 100, **When** пользователь отмечает "угадал", **Then** вес
   остается 100.
6. **Given** текущий вес слова равен 0, **When** пользователь отмечает "не угадал", **Then** вес
   остается 0.

### User Story 2 - Управление словами и папками (Priority: P2)

Как пользователь, я хочу добавлять слова в папки, редактировать и удалять их, чтобы управлять
личным словарем под разные контексты (например, конкретная книга) и выбирать, какие слова попадут
в общий набор карточек.

**Why this priority**: Качественное обучение невозможно без удобного ввода и организации слов.

**Independent Test**: Создать 2 папки и 5 слов, часть слов включить только в папки, часть в общий
набор. Проверить редактирование из карточки и профиля, удаление из профиля, а также выбор набора
карточек (общий или конкретная папка).

**Acceptance Scenarios**:

1. **Given** пользователь добавляет новое слово, **When** он выбирает папку и опцию "показывать в
   общем наборе", **Then** слово сохраняется в выбранной папке и попадает в общий набор только при
   включенной опции.
2. **Given** пользователь находится на экране карточек, **When** он переключает активный набор с
   общего на конкретную папку, **Then** карточки показываются только из выбранного набора.
3. **Given** слово есть в словаре, **When** пользователь редактирует его из карточки или из списка
   в профиле, **Then** везде отображается обновленное значение.
4. **Given** слово есть в списке профиля, **When** пользователь удаляет его, **Then** слово
   исчезает из всех наборов и не участвует в тренировке.

### User Story 3 - Профиль и проверка предложений (Priority: P3)

Как пользователь, я хочу видеть прогресс в профиле и проверять свои предложения с изучаемыми
словами, чтобы отслеживать регулярность и качество практики.

**Why this priority**: Статистика и обратная связь по предложениям повышают мотивацию и глубину
обучения.

**Independent Test**: Открыть профиль, проверить статистические показатели, задать ключ Gemini,
открыть модальное окно проверки предложения на карточке и получить результат проверки.

**Acceptance Scenarios**:

1. **Given** пользователь открывает профиль, **When** данные загружены, **Then** отображаются:
   общее количество слов, текущий daily streak, число слов, добавленных за текущую неделю, список
   папок и полный список слов.
2. **Given** пользователь впервые за день открыл приложение, **When** дата входа обновляется,
   **Then** daily streak увеличивается по правилам непрерывной серии.
3. **Given** пользователь ввел ключ Gemini в шапке профиля, **When** он сохраняет ключ,
   **Then** последующие проверки предложений используют этот ключ.
4. **Given** пользователь ввел предложение с изучаемым словом, **When** запускается проверка,
   **Then** пользователь получает либо ответ "ok", либо исправленную версию предложения.

---

### Edge Cases

- Пользователь пытается начать тренировку при пустом наборе карточек.
- В выбранной папке нет слов, подходящих для тренировки.
- Слово добавляется с уже существующей парой "слово-перевод".
- Пользователь удаляет слово, которое находится в текущей активной карточке.
- Устройство офлайн: локальные функции (просмотр, тренировка, редактирование, статистика) должны
  оставаться доступными, а проверка предложения должна показать понятную ошибку связи.
- Если часть локальных данных повреждена, система должна восстановить валидные данные и сбросить
  только поврежденный раздел, уведомив пользователя о восстановлении.
- Для проверки предложения сервис Gemini недоступен или превышен лимит: система должна
  автоматически выполнить fallback на альтернативный проверяющий сервис; если fallback также
  недоступен, показать понятную пользователю ошибку.
- Ключ Gemini отсутствует, недействителен или превышен лимит: пользователь должен получить
  понятное сообщение и возможность повторить попытку.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: Система MUST показывать карточки для изучения слов и выражений из выбранного набора.
- **FR-002**: Система MUST открывать обратную сторону текущей карточки по нажатию на карточку.
- **FR-003**: Система MUST начинать показ каждой новой карточки со слова или перевода с вероятностью
  50/50.
- **FR-004**: Пользователь MUST иметь возможность отметить результат ответа кнопками на карточке
  (угадал/не угадал).
- **FR-005**: Пользователь MUST иметь возможность отметить результат ответа свайпом вправо/влево с
  тем же эффектом, что и кнопки.
- **FR-006**: Система MUST увеличивать вес слова на 1 при результате "угадал".
- **FR-007**: Система MUST уменьшать вес слова на 3 при результате "не угадал".
- **FR-008**: Система MUST ограничивать вес слова диапазоном от 0 до 100.
- **FR-008a**: Система MUST использовать вес для определения частоты показа карточек в активном
  наборе: чем ниже вес, тем выше вероятность повторного показа карточки.
- **FR-009**: Система MUST хранить слова, папки, статистику и пользовательские настройки локально
  на устройстве между сессиями.
- **FR-009a**: При обнаружении поврежденных локальных данных система MUST пытаться выполнить
  частичное восстановление; при невозможности восстановления MUST сбрасывать только поврежденный
  раздел и уведомлять пользователя.
- **FR-010**: Система MUST предоставлять два экрана: экран карточек и экран профиля.
- **FR-011**: Экран профиля MUST отображать общее количество слов, daily streak, количество слов,
  добавленных за текущую неделю, список папок и полный список слов.
- **FR-012**: Пользователь MUST иметь возможность добавлять слово в конкретную папку при создании
  записи.
- **FR-012a**: Система MUST обеспечивать глобальную уникальность карточки по паре "слово+перевод";
  существующая карточка при повторном добавлении должна переиспользоваться, а не дублироваться.
- **FR-013**: При добавлении слова в папку пользователь MUST иметь возможность включить/выключить
  показ этого слова в общем наборе карточек.
- **FR-014**: Экран карточек MUST позволять выбрать обучаемый набор: общий или конкретная папка.
- **FR-014a**: Система MUST сохранять последний выбранный обучаемый набор (общий или конкретная
  папка) между перезапусками приложения.
- **FR-015**: Пользователь MUST иметь возможность редактировать слово из карточки и из списка слов
  в профиле.
- **FR-016**: Пользователь MUST иметь возможность удалять слово из профиля.
- **FR-017**: Карточка MUST предоставлять действие "проверить предложение", открывающее модальное
  окно с текстовым вводом.
- **FR-018**: После отправки предложения на проверку система MUST возвращать пользователю либо "ok",
  либо корректно составленный вариант предложения.
- **FR-018a**: Результат проверки предложения MUST иметь нормализованный формат:
  `status in {ok, corrected, error}` и текст `message` для отображения пользователю.
- **FR-019**: Экран профиля MUST позволять пользователю задать и изменить ключ Gemini.
- **FR-020**: Приложение MUST поддерживать темную и светлую тему и позволять пользователю
  переключаться между ними.
- **FR-021**: Интерактивные элементы MUST иметь понятные названия действий и обратную связь для
  состояний ожидания и ошибки.
- **FR-022**: Для сетевой проверки предложения система MUST показывать понятную пользователю ошибку
  при недоступности сервиса или проблеме с ключом, без блокировки остальных функций приложения.
- **FR-023**: Для проверки предложений система MUST сначала обращаться к Gemini, а при ошибках
  квоты/доступности/биллинга MUST автоматически выполнять fallback на LanguageTool; только при
  недоступности обоих сервисов допускается `error` результат.
- **FR-024**: Переключатель набора карточек (общий / папки) на экране карточек MUST отображаться в
  горизонтально прокручиваемой строке, если элементы не помещаются по ширине.
- **FR-025**: При добавлении нового слова выбор папки MUST выполняться через выпадающий список
  (не сетку чипов).
- **FR-026**: Табы «Слова» / «Папки» на экране профиля MUST занимать по половине ширины экрана и
  MUST NOT иметь отдельной карточной подложки вокруг строки табов (достаточно нижней границы у
  активного таба).
- **FR-027**: Кнопка добавления новой папки в профиле MUST отображаться без видимой обводки.
- **FR-028**: На вкладке «Папки» в профиле MUST быть поле поиска по названию папки (общая
  sticky-область поиска может использоваться для слов и для папок в зависимости от активной
  вкладки).
- **FR-029**: В списке папок профиля при нажатии на папку MUST раскрываться список слов,
  назначенных этой папке; повторное нажатие MUST сворачивать список.

### Key Entities *(include if feature involves data)*

- **Word Card**: Словарная карточка пользователя с полями "слово", "перевод", "вес",
  "дата добавления", связями с папками и признаком включения в общий набор. Поле "вес" определяет
  вероятность показа в тренировке (меньший вес = более частый показ).
- **Uniqueness Rule**: Пара "слово+перевод" является глобально уникальной во всем словаре; папки
  хранят связи с существующими карточками.
- **Folder**: Пользовательская категория слов (например, "книга"), объединяющая карточки для
  тематического обучения.
- **Training Set**: Активный набор для сессии тренировки (общий или выбранная папка).
- **User Stats**: Показатели прогресса: общее количество слов, текущий daily streak, дата последнего
  входа, количество слов за неделю.
- **Sentence Check Request**: Пользовательский ввод предложения с текущим изучаемым словом.
- **Sentence Check Result**: Результат проверки предложения: "ok" или исправленный текст, либо
  сообщение об ошибке проверки.
- **Theme Preference**: Выбор темы интерфейса (светлая/темная).
- **Gemini Key Setting**: Пользовательское значение ключа для проверки предложений.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Не менее 95% тестовых пользователей могут добавить первое слово, открыть карточку и
  выполнить одну оценку "угадал/не угадал" менее чем за 2 минуты после первого запуска.
- **SC-002**: В 100% проверенных сценариев вес слова никогда не выходит за границы 0..100.
- **SC-003**: Не менее 90% проверенных карточек в тестовой сессии стартуют со стороны
  слово/перевод в пропорции, не выходящей за диапазон 45/55-55/45 на выборке из 200 показов.
- **SC-004**: Не менее 95% локальных пользовательских данных (слова, папки, настройки, статистика)
  сохраняются между перезапусками приложения без потерь.
- **SC-005**: Не менее 90% пользователей в usability-проверке находят и переключают активный набор
  карточек (общий/папка) без подсказок.
- **SC-006**: Не менее 95% запросов проверки предложения возвращают пользователю результат или
  понятную ошибку в течение 10 секунд при стандартном сетевом подключении.
- **SC-007**: Не менее 95% взаимодействий с карточкой (тап для переворота, свайп/кнопка оценки)
  визуально подтверждаются пользователю менее чем за 100 мс на целевых устройствах.

## Assumptions

- Приложение ориентировано на одного локального пользователя устройства без регистрации и
  синхронизации между устройствами в рамках первой версии.
- Все основные функции обучения и управления словарем доступны офлайн; сеть требуется только для
  проверки предложений.
- Пользователь может создавать любое количество папок, и одно слово может принадлежать нескольким
  папкам.
- В рамках первой версии "количество слов за неделю" рассчитывается по дате добавления слова в
  словарь.
- Daily streak увеличивается один раз в день при первом открытии приложения и сбрасывается при
  пропуске хотя бы одного календарного дня.
